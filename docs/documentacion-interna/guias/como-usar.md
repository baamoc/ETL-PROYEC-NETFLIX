# Como usar esta documentacion interna

Esta carpeta esta pensada para que cualquier companero entienda el estado del proyecto rapido y sepa donde mirar antes de tocar algo.

## Si eres nuevo en el proyecto

1. Lee `docs/documentacion-interna/indice.md`.
2. Revisa `docs/documentacion-interna/seguimiento/resumen-avance.md` para entender el estado real.
3. Consulta `docs/documentacion-interna/reglas/decisiones-tecnicas.md` para no romper reglas base.
4. Usa `docs/documentacion-interna/guias/guia-colaboracion.md` para ver como trabajar en equipo.

## Si vas a trabajar en ETL

1. Revisa `docs/documentacion-interna/seguimiento/resumen-avance.md`.
2. Confirma reglas en `docs/documentacion-interna/reglas/decisiones-tecnicas.md`.
3. Mira el ultimo registro en `docs/documentacion-interna/seguimiento/registros/`.
4. Si avanzas un pipeline, deja configuracion, validacion y resultado documentados en el registro correspondiente.

## Si vas a actualizar documentacion

1. Mantene actualizado `docs/documentacion-interna/seguimiento/resumen-avance.md`.
2. Si el cambio es relevante, crea o actualiza un archivo en `docs/documentacion-interna/seguimiento/registros/`.
3. Si cambias reglas o criterios tecnicos, actualiza `docs/documentacion-interna/reglas/`.
